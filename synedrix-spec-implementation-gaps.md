## Summary: Severity Tiers

### 🔴 Critical Gaps (core spec promises with zero implementation)

1. **Planner & Focus Mode (§7.8)** — ✅ Implemented. `/planner` page with goals panel, next-best topic, overdue topics, session templates, recovery plan. `FocusMode` component with full-screen timer, pause/play, reflection capture. `SessionLauncher` component for starting tracked sessions. `sessionTemplates` table with full CRUD.
2. **Analytics & Insights (§7.10)** — No analytics page, no derived insights, no visualizations beyond basic stats.
3. **Observability stack (PostHog, Sentry, OpenTelemetry)** — Not integrated. AI-heavy app is close to undebuggable without this (§15).

### 🟡 Major Gaps (significant spec promises with minimal or no implementation)

4. **Goals system** — ✅ Now implemented. `convex/goals.ts` with full CRUD (list, create, update, increment, remove, getSnapshot) + `GoalCompletionSnapshot` component on dashboard. Daily and weekly goals have UI; the dedicated planner page is still TBD but the data layer and dashboard surface are complete.
5. **Personalization engine (§7.9)** — ✅ Implemented. `convex/tutorModes.getPersonalizationSignals` returns recommended session type, suggested revision timing, difficulty adjustment, and AI response-style selection — all driven from tutor profile + topic mastery/mistake data.
6. **Model routing by task (§12)** — Single model for all AI tasks. No fallback chain.
7. **Tests: integration + E2E (§15)** — Only ~5 unit test files. No integration tests, no Playwright E2E tests.
8. **Rich text editor (§9)** — Notes use plain textareas. No TipTap/Lexical.

### 🟢 Minor Gaps (nice-to-haves or partially implemented)

10. **Tests and mock exams (§7.2)** — ✅ Implemented. `SubjectTestsPanel` with grade distribution bar chart, graded run history, in-progress tracking, and per-run skills breakdown.
11. **Data export / account deletion (§16)** — Not implemented (links out to Clerk for now).
12. **Rate limiting + moderation (§12)** — Not implemented.
13. **Monorepo structure (§18)** — Single flat repo.
14. **TanStack Query / Zustand (§9, §11)** — Not used; Convex hooks suffice for current scale but spec promised them.
15. **Feature-first folder structure (§11, §17)** — Flat components/ layout instead of features/.
16. **Concise vs detailed AI toggle (§12)** — Not implemented.
17. **Physics lesson-level diagrams (§7.3)** — ✅ Implemented.
18. **Reduced motion support (§15)** — No prefers-reduced-motion checks.

### ✅ Phase Alignment Reality Check

The codebase has **fully implemented Phases 1–2** (Foundation + Core Study Loop) with excellent quality. Phase 3 (Planning and Intelligence) is approximately **70% complete** (planner page + focus mode + goals system + nextBest + personalization signals + weak-topic detection + recovery plan + session templates). Phase 4 (Polish) is approximately **25% complete** (weekly consistency + confidence comparison + lesson diagrams + gamification wins + resizable drawer + search/Cmd+K + timer + full IA surface coverage).
